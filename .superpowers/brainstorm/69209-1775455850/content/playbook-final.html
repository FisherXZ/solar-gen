Interaction Model

  • Left column ("Right Now") — dynamic, fed from the same stats the Briefing page uses. Click pre-fills chat with relevant prompt (e.g. "Let's triage the 61 pending reviews"). Subtle → /review link navigates to dedicated page.
  • Right column ("I can help you") — static outcome cards. Click pre-fills chat with a goal-oriented prompt (e.g. "I want to deep-dive a company"). Agent then asks follow-up ("Which company?").
  • Dynamic items hide when zero — if awaiting_review is 0, that nudge doesn't show. If all are zero, left column shows "You're all caught up" with a single subtle CTA.
  • Future: generative UI — when the agent handles review triage or contact discovery in chat, it renders interactive cards inline (approve/reject buttons, contact cards with LinkedIn links) so users don't need to tab-switch. This is a separate spec.
